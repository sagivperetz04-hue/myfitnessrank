MyFitnessRank leaderboards service deployed.

Release:   {{ .Release.Name }}
Namespace: {{ .Release.Namespace }}
Image:     {{ .Values.image.repository }}:{{ .Values.image.tag }}

The API is internal-only by default — the frontend nginx proxies /api/leaderboards/ to it.
On install/upgrade a Job seeds the OpenPowerlifting dataset into the DB (names encrypted
at rest); user lift rows (source='user') are never touched by the seed.

Check rollout:
  kubectl -n {{ .Release.Namespace }} rollout status deployment/{{ include "leaderboards.fullname" . }}

Read the board (from inside the cluster):
  kubectl -n {{ .Release.Namespace }} run curl --rm -it --image=curlimages/curl --restart=Never -- \
    curl -s "http://{{ include "leaderboards.fullname" . }}:{{ .Values.service.port }}/api/leaderboards?sex=M&sort=ratio"

Requires secrets in namespace {{ .Release.Namespace }} before pods become ready:
  "{{ .Values.db.existingSecret }}"         (keys: username, password, database)
  "{{ .Values.jwt.existingSecret }}"        (key: {{ .Values.jwt.secretKey }}) — shared with auth
  "{{ .Values.encryption.existingSecret }}" (key: {{ .Values.encryption.secretKey }})
